# Agate pipeline nodes

Developer guide for adding or changing **Agate graph nodes** end to end: worker runtime, graph builder UI, run output, and (when needed) processed-item review.

**Not Stylebook canonical types.** Most nodes emit pipeline JSON only. Canonical entity types (person, organization, work, location extract + substrate + Stylebook + review tabs) use [`ENTITY_TYPES.md`](ENTITY_TYPES.md) and [`.cursor/skills/add-entity-type`](../.cursor/skills/add-entity-type/SKILL.md).

**Planning net-new nodes:** use [`.cursor/skills/add-agate-node`](../.cursor/skills/add-agate-node/SKILL.md) — interview → `prd/<slug>/prd.md` → implementation issues.

**Panel UI patterns:** [`FRONTEND.md`](FRONTEND.md) → **Agate nodes and node panels** (typography, tabs, sync, `nodePanelAiModel`).

## Node profiles

Branch implementation and review work by profile. Every node folder declares a profile in its PRD; `metadata.json` `category` should align.

| Profile | Role | Backfield examples | Stylebook / review |
|---------|------|-------------------|-------------------|
| **Input** | Ingress text, JSON, or S3 batch | `TextInput`, `JSONInput`, `S3Input` | None |
| **Output** | Egress JSON or substrate persist | `Output`, `DBOutput` | `DBOutput` persists via worker; review only via downstream entity tabs |
| **Extract** | LLM or rules extraction | `PlaceExtract`, `PersonExtract`, `OrganizationExtract` | **Canonical entity extract** → [`add-entity-type`](../.cursor/skills/add-entity-type/SKILL.md). Non-canonical extract stays here |
| **Enrich** | Transform or resolve upstream rows | `GeocodeAgent` | May **feed entity review** (e.g. Places tab) without a new canonical type |
| **Embed** | Vector / semantic indexing | `EmbedText`, `EmbedImages` | Embed Text: Info tab status; Embed Images: **Images** review tab (read-only) |
| **Other** | Flow control, gather, stats, format | `Gather` | Panel + JSON (Gather also shows Output tab preview) |

## Persistence and substrate

Most nodes **only emit JSON** into `node_outputs` / consolidated payloads. Durable writes normally flow **`… → DBOutput → worker/substrate/orchestration.py`**, keyed by consolidated domains (`places`, `people`, `organizations`).

Ask substrate / schema questions when **any** of these apply:

| Trigger | Typical work |
|---------|--------------|
| **Durable cross-run storage** | New `substrate_*` or `agate_*` table + Alembic migration ([`backfield-db-change`](../.cursor/skills/backfield-db-change/SKILL.md)) |
| **New consolidated key for `DBOutput`** | Orchestration handler or entity registry — often overlaps **Extract + entity type** |
| **Reads/writes existing substrate** | Document tables (e.g. geocode cache, `substrate_article` fields); usually no new migration |
| **Pass-through only** | Skip substrate section |

Substrate decisions to record in the PRD:

- **Durability scope** — run-scoped JSON vs article-scoped vs project-scoped
- **Table namespace** — `substrate_*` (shared domain) vs `agate_*` (pipeline artifact)
- **Persist path** — node writes directly vs emits JSON for downstream `DBOutput`
- **Stylebook FK** — any canonical link, or pipeline-only

## Agate review tiers (non-entity nodes)

Processed-item detail tabs today: `info`, `places`, `people`, `organizations`, `images`, `meta`, `json` (`apps/agate-ui/src/lib/review/content/detailTab.ts`). Dedicated editorial review exists for entity lanes only.

| Tier | Use when | Work |
|------|----------|------|
| **None** | Diagnostic or pass-through | No review code |
| **Panel + JSON** | Default for Enrich / Embed / Other | Node panel Output tab + item **JSON** tab |
| **Feeds entity review** | Output merged into `places` / `people` / `organizations` | Document merge contract; may touch `agate-api` overlay / merge helpers and existing review lib — **no new tab** unless product requests |
| **New review surface** | Human edits this node's rows in dedicated UI | Out of scope unless PRD explicitly requests |

## Checklist: net-new node

Use after a PRD exists (via [`add-agate-node`](../.cursor/skills/add-agate-node/SKILL.md)) or when porting a known node from agate-ai-platform.

### 1. Runtime (issue 01)

1. Create `packages/backfield-agate/src/agate_nodes/<snake_case>/`:
   - `metadata.json` — `type`, `label`, `icon`, `category`, `description`, `inputs`, `outputs`, `defaultParams`, optional `requiredUpstreamNodes`
   - `node.py` — `run_<snake>(params, inputs) -> dict`
   - Optional `runner.py`, `prompts/`, Pydantic schemas
2. Add `agate_runtime/nodes/<snake_case>.py` re-exporting `run_*`.
3. Register in `NODE_RUNNERS` (`agate_runtime/nodes/__init__.py`). Add to `ASYNC_NODE_RUNNERS` if async.
4. Add unit tests for output shape and error cases.
5. For **LLM nodes**: ensure worker LLM tracking gets `metadata.json` **`label`** for step display names.

### 2. Graph UI (issue 02)

1. Add `ui/NodeComponent.tsx` and `ui/PanelComponent.tsx` under the node package.
2. Update `apps/agate-ui/src/lib/nodePanelTabs.ts` for tab ids.
3. Extend `nodeUtils.ts` `iconMap` if needed.
4. Set `requiredUpstreamNodes` / port ids in metadata; verify `nodeCompatibility.ts` behavior.
5. Run from `apps/agate-ui`:

   ```bash
   npm run sync-nodes
   ```

6. Commit **both** package tree and synced `apps/agate-ui/src/nodes/` + `registry.ts`.

See [`FRONTEND.md`](FRONTEND.md) → **Agate nodes and node panels** for panel design system, `NodePanelTabGate`, and AI model selects.

### 3. Run output / visualization (issue 03 — skip when N/A)

1. Wire panel Output tab via `currentRun` + `getNodeOutputById`.
2. Optional `ui/VisualizationComponent.tsx` for run detail (`apps/agate-ui/src/lib/visualizations.ts`).
3. Vitest for tab routing if non-default (`nodePanelTabs.test.ts`).

### 4. Review merge hook (issue 04 — skip when tier is None or Panel+JSON only)

1. Document which consolidated key and merge path the node feeds.
2. Implement in `agate-api` processed-item merge/overlay only when feeding an **existing** entity tab.
3. Add or extend `apps/agate-ui/src/lib/review/` helpers if the merge surface changes.

### 5. Smoke / demo (issue 05)

1. Document a minimal graph (or extend local bootstrap graph) that exercises the node.
2. Run `make lint` and `make test`.
3. Run `make smoke` when behavior crosses worker, API, or live stack boundaries.

## Checklist: adjust existing node

No PRD required. Follow [`FRONTEND.md`](FRONTEND.md) checklist when UI/metadata changes:

1. Edit package source under `agate_nodes/<snake>/` (not synced app copies).
2. Run `npm run sync-nodes` when metadata or `ui/` changed.
3. Run targeted tests; `make lint` / `make test` before merge.

Skip runtime, review, and schema work unless the change alters output shape, ports, LLM behavior, or persistence.

## Profile-specific notes

### Input

- Bookend types: `TextInput`, `JSONInput`, `S3Input` (`flowValidation.ts`).
- `S3Input` batch graphs use `execute_s3_batch_setup`; inline graphs use single `agate_processed_item` at run create — see [`API.md`](API.md).

### Output

- `Output` — consolidated JSON view only.
- `DBOutput` — worker-local persist (`apps/worker/src/worker/nodes/db_output.py`); stub in package for offline runs.

### Extract (non-canonical)

- Follow LLM prompt layout in [`ENTITY_TYPES.md`](ENTITY_TYPES.md) → **Agate nodes** (`prompts/extract.md`, `{text}` last).
- If extract rows should land in Stylebook substrate, stop — use **`add-entity-type`**.

### Enrich

- Often sits between extract and output bookend (e.g. `PlaceExtract → GeocodeAgent → DBOutput`).
- May read worker env (`BACKFIELD_PROJECT_ID`) and Stylebook cache bundles — see [`ARCHITECTURE.md`](ARCHITECTURE.md) geocode cache section.
- **Article Metadata (`ArticleMetadata`)** — LLM assigns one preset-driven tag (e.g. topic category) from article text; emits consolidated `article_metadata` persisted by **DBOutput** to `substrate_article_meta`. Processed-item **Meta** tab allows editing **category** only (free text). Smoke: `make smoke-article-metadata` (in-process, mocked LLM) or `make smoke-article-metadata-stack` (live stack; create an **Article Metadata starter** graph or set `SMOKE_ARTICLE_METADATA_GRAPH_ID`).

### Embed

- Clarify vector storage (new table vs existing semantic indexing in `backfield_entities.ingest.semantic_indexing`).
- **Embed Text (`EmbedText`)** stores one vector per article in `substrate_article_embedding`, emitted as consolidated `article_embedding` and persisted by **DBOutput** (honors reconciliation policy). Processed-item **Info** tab shows a compact indicator; full payload appears in the **JSON** tab.
- Default review tier: Panel + JSON (Info indicator only).

### Other

- Explicitly document port compatibility and whether the node mutates upstream keys in place vs emits a new key.
